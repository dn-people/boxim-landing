# GEO 측정 루프 — 월간 AI 응답 재측정 운영 문서

- **작성일**: 2026-07-15
- **대상**: `dn-people.com`(랜딩) + `dnbn.co.kr`(공식 서비스) — 브랜드 "동네방네"(운영사 주식회사 동네사람들)를 하나의 엔티티로 묶어 측정
- **배경 문서**: `docs/geo/GEO_GAP_ANALYSIS.md` 3장(측정 루프 부재 지적), `docs/geo/P0_DNBN_ENTITY_SPEC.md`(측정 대상이 될 구조 변경 예시)
- **이 문서가 정의하는 것**: 구조적 AEO/GEO(JSON-LD, robots.txt, llms.txt 등)는 이미 구현돼 있다. 없는 것은 **"AI가 실제로 뭐라고 답하는지"를 고정 질문셋으로 월 1회 재측정하는 운영 루프**다. 본 문서는 그 루프의 절차·질문셋·기록 양식·요약 지표를 확정한다.

---

## 1. 목적

월 1회, 고정된 질문셋을 동일한 조건으로 ChatGPT / Google Gemini / Perplexity에 던져 다음 세 가지를 재측정한다.

1. **브랜드 언급(Mention)** — AI 답변에 "동네방네/dnbn/dn-people"이 언급되는가
2. **출처 인용(Citation)** — 답변의 출처 링크로 dn-people.com 또는 dnbn.co.kr이 실제로 인용되는가
3. **정체성 정확도(Accuracy)** — AI가 동네방네를 무엇을 하는 회사로 설명하는가, 오인은 없는가

그리고 P0(dnbn.co.kr Organization JSON-LD 적용)처럼 구조를 바꾸는 작업이 있을 때, **그 작업 전/후의 측정치를 같은 조건에서 비교**해 실제로 AI 응답이 달라졌는지 확인한다. 구조 검증(`scripts/verify-aeo.js`)이 "규격을 맞췄는가"를 보는 것과 달리, 이 루프는 "그래서 AI가 실제로 뭐라고 답하는가"를 본다.

## 2. 측정 대상 엔진 (3종)

| 엔진 | 조건 |
|---|---|
| ChatGPT | 웹 검색을 켠 상태, GPT 최신 모델 사용 |
| Google Gemini | 기본 웹 검색 동작 상태 |
| Perplexity | 기본 검색 동작 상태 |

**원칙**

- 로그인/개인화는 최소화한다 — 가능하면 시크릿(비로그인) 모드로 접속해 과거 대화·개인화 이력이 답변에 섞이지 않게 한다.
- 매월 같은 조건으로 측정한다 — 검색 기능 켬/끔, 로그인 여부가 회차마다 달라지면 비교가 무의미해진다.
- 각 회차 기록 파일 상단에 그 회차에 실제로 사용한 조건(검색 켬/끔, 로그인 여부, 사용 모델명)을 반드시 남긴다. 조건이 전월과 달라졌다면 그 사실 자체를 비고에 명시한다.

## 3. 고정 질문셋

아래 두 묶음(A: 브랜드형 10개, B: 비브랜드형 10개, 총 20개)을 **매월 동일한 문구로 그대로 사용**한다. 질문 문구를 임의로 바꾸지 않는다. 질문을 추가/변경해야 하면 버전을 올리고(v2 등) 이전 회차와의 비교 시 "질문 문구가 달라졌다"는 점을 반드시 주석으로 남긴다.

## 4. 질문별 기록 지표

측정 회차마다 **(질문 × 엔진)** 조합, 즉 20문항 × 3엔진 = 60개 셀 각각에 대해 아래 4개 항목을 기록한다.

| 지표 | 값 | 설명 |
|---|---|---|
| **Mention** | Y / N | 답변 본문에 "동네방네" 또는 "dnbn" 또는 "dn-people"이 언급되는가 |
| **Citation(URL)** | 실제 URL 또는 `-` | 출처(각주·링크·참고자료)로 dn-people.com 또는 dnbn.co.kr이 인용됐는가. 인용됐다면 실제 URL을 그대로 적고, 없으면 `-` |
| **정확도(1~3)** | 1 / 2 / 3 | 3=정체·사실이 정확, 2=부분 정확하거나 모호, 1=오인·오답. 브랜드형(A군)에서 특히 중요 — 예: "동네방네를 통신사로 오인", "운영사를 다른 회사로 서술" 등은 1점 |
| **비고** | 자유 서술 | 오인 내용, 경쟁사가 대신 인용됐는지, 눈에 띄는 표현(예: "리퍼폰 취급점"처럼 부정확하거나 애매한 요약) 등 |

비브랜드형(B군)은 애초에 브랜드를 직접 묻지 않으므로 Mention이 N이어도 정상이다. B군에서 중요한 건 Citation과, Mention이 있을 경우의 정확도다.

## 6. 월간 요약 지표

각 회차 파일(`YYYY-MM.md`) 맨 위에 아래 4가지를 집계해 적는다. 상세 표(§5)를 다 채운 뒤 계산한다.

| 지표 | 계산식 |
|---|---|
| **브랜드 언급률** | (A군에서 Mention = Y인 셀 수) ÷ (브랜드형 10문항 × 엔진 수 3 = 30) |
| **인용률** | (전체 60셀 중 Citation이 `-`가 아닌 셀 수) ÷ 60 — 필요하면 A군/B군을 나눠서도 별도 표기 |
| **평균 정확도(브랜드형)** | A군 30셀의 정확도(1~3) 평균 |
| **엔티티 정확도 특이사항** | 특히 Gemini가 동네방네의 정체를 오인하는지(A1·A2·A4·A7·A9 위주로 확인) — **전월 대비 무엇이 달라졌는지**를 한두 문장으로 서술 |

이 4개 지표는 회차 간 비교의 핵심 신호다. 특히 "엔티티 정확도 특이사항"은 P0(dnbn.co.kr 엔티티 신호 비대칭 해소) 같은 구조 변경이 실제로 Gemini의 오인을 고쳤는지 확인하는 항목이므로, 전월 값을 반드시 옆에 같이 적어 비교한다.

## 8. 운영 규칙

- **실행 주기**: 매월 1회(권장: 매월 첫 주). 담당자와 실제 실행 날짜를 해당 회차 파일 상단에 기록한다.
- **질문 문구·조건 고정**: §3의 20문항 문구를 임의로 바꾸지 않는다. 추가·변경 시 버전(v2 등)을 표기하고, 버전이 다른 회차끼리 비교할 때는 "질문 문구가 달라졌다"는 점을 반드시 주의사항으로 남긴다.
- **dn-people.com 단독이 아니라 dnbn.co.kr과 묶어서 측정한다**: 두 도메인은 같은 브랜드 엔티티(동네방네)이므로, 브랜드형 질문(A군)의 답변에서 두 도메인 중 어느 쪽이 언급/인용되는지, 그리고 P0 적용 전/후로 이 비대칭이 해소됐는지를 함께 확인해야 한다. 한쪽만 보면 엔티티 신호 문제를 놓친다.
- **회차 파일 위치**: `docs/geo/measurements/YYYY-MM.md`. 매 회차는 `docs/geo/measurements/TEMPLATE.md`를 복사해서 만든다(파일명만 해당 측정월로 변경).
- **실제 측정 없이 답변을 지어내지 않는다**: 각 엔진에 실제로 질문을 던져 받은 응답만 기록한다. 확인이 안 된 셀은 비워 두거나 "미측정"으로 표기하고, 임의로 채우지 않는다.
